# 扩展提示语和动画效果 - 设计文档

## 需求

用户要求：

1. 增加更多的正确和错误提示语，语言要活泼可爱，紧跟最近网络用语潮流
2. 提示语弹出的动画效果多增加一些

## 现状

- 正确提示语 `RIGHT_RESPONSES`: 16条
- 错误提示语 `ERROR_RESPONSES`: 4条
- 动画类 `ANIMATION_CLASSES`: 19种
- 架构：三个常量数组，答题完成后随机抽取一个

## 设计决策

### 提示语扩展

| 类别       | 当前 | 目标 | 风格                                               |
| ---------- | ---- | ---- | -------------------------------------------------- |
| 正确提示语 | 16   | ~35  | 紧跟潮流，加入热门网络用语，保持活泼可爱适合小朋友 |
| 错误提示语 | 4    | ~18  | 鼓励为主，轻松可爱，不让小朋友受挫                 |

**正确提示语新增方向**:

- "太酷啦！"
- "这题直接拿捏了"
- "秒杀！毫不费力"
- "简直yyds！"
- "开了挂一样！"
- "这题超简单的好吧"
- "这波操作666"
- "表现绝佳！"
- "根本难不倒你"
- "赞爆了！"
- "太丝滑了"
- "一个字，绝"
- "实力爆棚"
- "答题小能手就是你"

**错误提示语新增方向**:

- "差一丢丢哦，再试试？"
- "没关系，再来一次！"
- "再仔细看看哦"
- "加油加油，你可以的！"
- "答错不灰心，下次一定对"
- "靠近了，再想想"
- "换个思路试试看？"
- "不要紧，再算一遍"
- "多想想，你可以的"

### 动画扩展

- 当前：19种
- 目标：增加到 **30种**（新增 11种）
- Animate.css 版本：v4.1.1
- 筛选原则：选择适合文字弹窗的强调/入场动画，避免位移过大影响布局的动画
- **新增动画列表**:
  - `animate__fadeIn`
  - `animate__fadeInDown`
  - `animate__fadeInUp`
  - `animate__fadeInLeft`
  - `animate__fadeInRight`
  - `animate__zoomInUp`
  - `animate__slideInDown`
  - `animate__slideInLeft`
  - `animate__slideInRight`
  - `animate__bounceIn`
  - `animate__bounceInUp`

添加后总共 30种，满足"30种左右"的需求。

### 架构保持不变

继续使用现有的常量数组 + 随机抽取模式：

- `RIGHT_RESPONSES`: 正确提示语数组
- `ERROR_RESPONSES`: 错误提示语数组
- `ANIMATION_CLASSES`: 动画类名数组
- 现有随机抽取逻辑 `getRandomElement` 保持不变
- 零侵入修改，不影响现有功能

## 代码修改范围

只需要修改 `index.html` 中三个常量数组的定义，其他代码不动。

## 验收标准

1. 正确提示语达到约35条，错误提示语达到约18条
2. 动画效果达到约30种
3. 语言风格活泼可爱，包含热门网络用语
4. 原有功能完全保留，不改变交互逻辑
